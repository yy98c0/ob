## 推荐本地完整验收流程

```bash
cd "$(git rev-parse --show-toplevel)"

# 1. 从 Obsidian 拉取要测的笔记（示例）
npm run pull:note -- 测试.md

# 2. 构建并自动检查 TikZ / 页面
npm run check:publish

# 3. 浏览器预览
npm run dev
# → http://localhost:8080/测试-渲染
# → http://localhost:8080/测试
```

## 上线前另需（一次性）

1. GitHub 仓库 + Pages（Actions）— 见 [操作指南.md](./操作指南.md)
2. `quartz.config.yaml` 中 `baseUrl` 改为真实 `用户名.github.io/仓库名`
3. Obsidian 安装 **Quartz Syncer**，Token + 仅标记发布
4. **不要**对 `个人博客/任务书.md` 执行 Mark / Publish

## 与 jz 全量 cp 方案对比

[jz 教程](https://jz-quartz.pages.dev/4.技术记录/4.同步obsidian文件到quartz) 仍可用，但会同步整个文件夹。当前 **Quartz Syncer（选择性发布）+ 本地 pull 脚本** 更符合任务书「Vault 独立、按钮发布」。
