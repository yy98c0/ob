# 个人学术博客 · 通俗操作指南

## 先搞懂三件事

| 东西 | 是什么 | 放哪 |
|------|--------|------|
| **Obsidian** | 你写笔记的私人本子 | `/Users/yy/Documents/Obsidian Vault` |
| **Quartz 站点** | 把笔记变成网页的「印刷厂」 | 本仓库（项目根目录） |
| **GitHub** | 网上存代码 + 自动帮你发布成网站 | 需要你注册账号并建一个仓库 |

**日常关系：**

- 在 Obsidian 里写文章（公式、TikZ 图都写在这里）
- 点「发布」→ 文章进 GitHub 仓库的 `content/` 文件夹
- GitHub 自动构建 → 全世界能打开你的博客网址

Obsidian 和 Quartz 文件夹**分开**，不会把你的草稿、任务书混进网站（只要你别误点「发布」）。

---

## 第一部分：第一次把网站弄上线（只做一次）

### 步骤 1：确认电脑环境

打开「终端」，输入：

```bash
node -v
npm -v
```

- Node 要 **22 或以上**
- npm 要 **10.9 或以上**

不够就去装 [Node.js 官网 LTS](https://nodejs.org/)。

---

### 步骤 2：在本地装好博客项目

```bash
cd "$(git rev-parse --show-toplevel)"
npm install
npx quartz plugin install --from-config
npm run build
```

**什么意思：** 下载依赖、安装 Quartz 插件、在本地生成一版网页（在 `public/` 文件夹）。

**成功标志：** 最后一行类似 `Done processing`，没有红色报错。

---

### 步骤 3：本地先看看长什么样

```bash
npm run dev
```

浏览器打开：**http://localhost:8080**

- 首页：`index`
- 测试公式和 TikZ：`测试-渲染`

看完在终端按 `Ctrl + C` 停掉服务。

---

### 步骤 4：在 GitHub 建一个空仓库

1. 登录 [github.com](https://github.com)
2. 右上角 **+** → **New repository**
3. 名字例如：`ob`（自己记得住就行；当前仓库为 [yy98c0/ob](https://github.com/yy98c0/ob)）
4. 选 **Public**
5. **不要**勾选「Add README」（本地已有代码）
6. 点 **Create repository**

记下你的用户名，例如 `zhangsan`，以后网址大概是：  
`https://zhangsan.github.io/your-repo/`

---

### 步骤 5：把本地项目推到 GitHub

在 GitHub 新建仓库页会有一串命令，也可以用下面（把 `YOUR_USERNAME` 和仓库名改成你的）：

```bash
cd "$(git rev-parse --show-toplevel)"
git add .
git commit -m "初始化学术博客"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/your-repo.git
git push -u origin main
```

第一次 push 可能要登录 GitHub（浏览器或令牌）。

---

### 步骤 6：打开 GitHub Pages（自动发网站）

1. 打开你的仓库 → **Settings** → 左侧 **Pages**
2. **Build and deployment** → **Source** 选 **GitHub Actions**
3. 回到仓库 **Actions** 标签，看 **Deploy to GitHub Pages** 是否在跑
4. 绿色勾 = 成功；点进去可看日志

等 1～3 分钟，访问：`https://YOUR_USERNAME.github.io/your-repo/`

---

### 步骤 7：改网站地址配置（很重要）

用编辑器打开项目根目录下的 `quartz.config.yaml`

找到 `baseUrl`，改成（**不要**写 `https://`）：

```yaml
baseUrl: YOUR_USERNAME.github.io/your-repo
```

保存后执行：

```bash
cd "$(git rev-parse --show-toplevel)"
npx quartz sync
```

**什么意思：** 站内链接、RSS 等都要知道「正式网址是什么」，改完再推一次 GitHub。

---

## 第二部分：在 Obsidian 里连接发布（只做一次）

### 步骤 8：安装 Quartz Syncer 插件

1. 打开 Obsidian，库选：**Obsidian Vault**（整个库）
2. **设置** → **社区插件** → 关闭「安全模式」（若还没关）
3. **浏览** → 搜索 **Quartz Syncer** → 安装并启用
4. 按插件向导填：

| 填什么 | 建议 |
|--------|------|
| GitHub Token | 见下面步骤 9 |
| 仓库 | `YOUR_USERNAME/your-repo`（当前：`yy98c0/ob`） |
| 内容目录 | `content` |
| 发布方式 | **只发布你标记过的笔记** |

---

### 步骤 9：申请 GitHub Token（给 Obsidian 用）

1. GitHub → 头像 → **Settings** → **Developer settings** → **Personal access tokens**
2. **Fine-grained** 或 **Classic** 均可；Classic 勾 **`repo`** 权限即可
3. 生成后**复制保存**（只显示一次）
4. 粘贴到 Quartz Syncer 设置里

**注意：** Token 像密码，不要发给别人、不要提交到笔记里。

---

### 步骤 10：安全习惯（必看）

- **`个人博客/`** 里的任务书、需求文档：**不要**标记发布
- 私密草稿：**不要**标记发布
- 只有想公开的文章才标记发布

站点配置里已忽略 `个人博客` 文件夹，但 Obsidian 若强行标记发布仍可能推到 GitHub，所以以「只标记要公开的文」为准。

---

## 第三部分：以后每次发文章（天天用）

### 写一篇新笔记

1. 在 Obsidian 正常写 Markdown
2. 公式用 `$...$` 或 `$$...$$`
3. TikZ 图用代码块，语言选 **tikz**：

````markdown
```tikz
\begin{tikzpicture}
  \draw (0,0) circle (1);
\end{tikzpicture}
```
````

   详细原理、排错与宏包说明见 **[TIKZ-QUARTZ.md](./TIKZ-QUARTZ.md)**。

4. **保存**（Cmd + S）

---

### 发布到网站

1. 命令面板（Cmd + P）搜：**Quartz Syncer: Mark**（标记当前笔记要发布）  
   - 或插件侧边栏里点「标记」
2. 再搜：**Quartz Syncer: Publish**（真正上传）
3. 打开 GitHub → **Actions**，等部署变绿
4. 浏览器刷新你的博客网址

**全程不用**打开终端复制文件；Syncer 会直接改 GitHub 上的 `content/`。

---

### 可选：一键发布快捷键

Obsidian 1.12+ 可在设置里给命令 **Quartz Syncer: Publish** 绑快捷键。

---

### 改样式、改 TikZ 逻辑时（偶尔）

这类改的是「印刷厂」本身，在本地项目里做：

```bash
cd "$(git rev-parse --show-toplevel)"
# 改 quartz/styles/custom.scss 或 quartz/plugins/transformers/tikz.ts 等
npm run dev    # 本地预览
git add .
git commit -m "更新样式"
git push
```

**正文内容**仍建议在 Obsidian + Syncer 发布；**外观和插件**在本仓库（项目根目录）里改。

## 第四部分：出问题怎么查

| 现象 | 可能原因 | 怎么办 |
|------|----------|--------|
| 本地 `npm run build` 报错 | 依赖没装全 | 再执行 `npm install` 和 `npx quartz plugin install --from-config` |
| 网站 404 | Pages 没开 Actions / 还在构建 | 看 GitHub Actions 是否成功 |
| 链接错乱 | `baseUrl` 没改对 | 按步骤 7 改 `quartz.config.yaml` 并 sync |
| 公式正常、TikZ 不显示 | TikZ 语法错 | 看终端 build 时 `[TikZ]` 报错；先本地 `npm run build` |
| Obsidian 发布失败 | Token 过期或仓库名错 | 在 Syncer 设置里 **Test connection** |
| 任务书出现在网上 | 误标记发布 | 在 Syncer 里取消标记并从远端删除（慎用 sync force） |

```
        ↓

```

## 常用命令速查

```bash
cd "$(git rev-parse --show-toplevel)"

npm run dev      # 本地预览
npm run build    # 只构建，不启动服务器
npx quartz sync  # 和 GitHub 同步配置/内容（改配置后用）
git pull         # Syncer 发布后，本地想预览最新文章时先拉取
```
